jQuery事件绑定的最佳实践

  • 2015-11-17
  • 作者：admin
  • 阅读(851)

在过去的几年里, AngularJS 已成为开发单页面的 web 应用程序的首选堆栈(温泉)。作为一个非常先进的 JavaScript 框架,它不仅最大程度地增强了 HTML 还简化了从设计 UI 到测试应用程序。

根据Libscore在线JS脚本库收集统计平台的统计，现在有超过9000的网站,包括一些世界上最著名的公司比如Intel, Sprint, NBC, Walgreens and ABC News,等都在使用Angular.

这就证明了Angular在开发者中间是非常流行的。读了本文你会明白开发者更喜欢学习和使用AngularJS的理由。

1) Google开发并且维护

由于AngularJS是有Google开发并且维护的，所以作为一名开发者你完全可以信赖并且在项目中使用它。AngularJS并不像一些Javascrip应用框架，是由一些独立的爱好者开发出来的，它是由天才的谷歌工程师Adam Abrons 和 Miško Hevery 共同创建的。这也是开发者们更喜欢使用AngularJS而不是Ember.js 和 Knockout.js的原因。

2) MVC架构

AngularJS利用流行的MVC(模型-视图-控制器)软件体系结构模式,而不是传统意义上MVC。一般来说,开发人员都需要应用程序分割成各种MVC组件,然后编写的所有基本代码连接在一起。这不是Angular的问题,它只是要求应用程序本身并处理一切。换句话说,Angular非常接近MVVM设计模式(Model-View-Viewmodel)。

3) 自定义指令

在Angular允许开发人员编写自定义,独立和可重用HTML组件,它们可以使用这么组件隐藏复杂的DOM结构,处理所有DOM操作以及定义一个特定的视图组件的行为应该如何表现。因为包含了所有DOM操作代码,你可以分离它们从你的MVC应用程序然后只修改展示数据。页面的展示效果是被定义到组件里面的。

4) 双向数据绑定减少了大量的模板的使用

双向数据绑定能够随着模型的变化自动更新视图，这是AngularJS独特并且强大的特性。传统的MVC框架要求开发者手动的存储数据到模型中，然后在视图中替换，这样操作会产生模板代码。与之不同的是Angular的双向数据绑定是讲模型绑定到HTML元素中，这样避免开发者编写不必要的代码。

5) 灵活的过滤器

过滤器和组件一样，它也是独立的功能，主要负责在数据到达视图前的拦截过滤。 它们从你的应用程序中分离出来，进行操作比如：改变一个文本字符串、实现分页和格式化小数点后一个数字。你可以在服务、控制器、指示和视图模板中使用它们。它们甚至可以对一个HTML表格进行排序而不使用任何JavaScript代码。

6) 优秀的依赖管理

Angular充分利用依赖注入(DI)来处理应用程序应该如何连接。这意味着,你不再需要使用不易维护的main方法来执行你的代码。通过添加依赖Angular立即为你提供它。Angular的依赖注入降低了服务器的负担，从而是你的应用更轻量级，响应速度更快。

7) 数据模型是POJO

Angular与其他框架不同的地方是，它的数据模式是POJO,也就意味着意味着你永远不需要使用任何额外的附加getter和setter函数通过存在的数据源集成到Angular中。这将使你的代码从模板中解放出来，能够方便的进行测试、维护以及重用。作用域的命名使Angular的模型与传统的模型以及依赖控制器获取数据有很大的不同。

8) 声明式UI

为了定义应用界面，Angular利用HTML（一种声明式程序语言）。与javascript中写的接口不同，HTML生成了更简单、更直观的用户界面。另外、HTML允许您为每个UI元素声明应该使用哪个控制器,这样你可以更专注于应用程序的用户界面的外观和行为。简单的定义你想要的，Angular会解决所有的依赖。

9) 平行开发

在大项目中通常会包含多样的关键性操作，Angular允许开发者单独的编写和测试代码。从应用程序中分离的指令能够使开发者在同一时间内在相同的代码上工作。开发者甚至可以重用别的开发者的代码而不会带来整合问题。这加速应用程序开发流程，从而在很大程度上,让开发人员的生活变得更轻松。

10) 上下文发布系统(PubSub)

虽然大多数网络上PubSub实现不是上下文的关系,但Angular中的PubSub系统却是上下文关系。PubSub系统是一个非常好的工具,它减弱了父节点和子节点之间的通信。Angular让通过broadcast()和emit() 使得PubSub系统更加的直观。当broadcast()发送一条消息广播给所有的子控制器,另一方面,emit()发送一条消息到所有的父控制器。另外,您还可以使用数据绑定来与控制器通信。

11) 建立可测试性

由于JavaScript是一种动态语言,Angular团队已经强制要求任何JavaScript代码通过健壮的多个测试。对于端到端的测试,Angular使用一个工具叫“量角器”——一个预构建端到端测试运行器。您还可以通过单元测试去测试更小的可测试应用程序组件。此外,Angular提供AngularJS Batarang扩展让开发人员调试他们的应用程序在Chrome浏览器。

12) 写更少的代码

上面提到的各种特性描述只为了一件事儿——你需要写更少的代码，就能完成一些功能。不管是视图或指令或双向数据绑定或任何其他功能,他们都节省开发人员编写自己的MVC管道。在一个典型的web应用程序有近80%的代码用于不同的dom的功能,Angular会自动清理所有不必要的应用程序代码。

13) 更容易使用

在一个Angular应用程序中，你只需要添加在HTML标签中添加一些属性。你甚至可以在本地系统上使用Angular,绝对不需要任何web服务器。你只需要有web浏览器和文本编辑器就可以开始工作了。此外,Angular团队提供了步进式的教程来帮助你在几分钟内就可以运行您的web应用程序。

14) 强大的社区支持

Angular拥有一个蓬勃发展，非常活跃开发者社区作为支撑,这意味着你会查询得到所有关于Angular的回答。社区不仅致力于角项目还开发兼容库,而且使其更加强大。这就是用Angular构建的应用程序产生更好的结果的原因。此外,有很多好地方——比如LearnAngular——在那里你可以在短的时间内学习Angular

15) 未来前景

正如你可能已经知道,Angular团队最近发布了一个全新版本的角,称为Angular2.0。根据官员,版本2.0是一个完整的重写有了巨大变化,最新集成与ECMAScript 6(ES6),综合路由、分层的依赖注入,可变路由等等。这证明Angular团队一直在为使它成为世界上最强大的和健壮的JavaScript框架而努力。
